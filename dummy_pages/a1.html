キーボードイベントを特定要素に固定させる方法はあるか？ †

キーボードイベントは特定要素に対して付される。それは、フォーカスされている要素に対してのキーボードイベントを設定する意図だと思われる。
stage.focusでフォーカスされている要素を強引に変えられるので、それを親要素のクリックイベント伝播させたりしてクリック時のフォーカス変更を抑止することはできるし、また、ENTER_FRAMEで毎フレーム書き換えても良いが、キーボードイベントだけでない要素のフォーカスが固定されてしまうことが不安。
どうにか、キーボードイベントを特定の要素に固定することをしたい。

サブクラス内でスーパークラスのプロパティを上書きするときに権限指定が必要 †

スーパークラスでこう定義しているプロパティを

private var hoge:* = 0;

サブクラスで定義しなおしたい場合は、こう（private付きで）書かないとダメっぽい

private var hoge:* = 1;

できれば、こう書いて、子が親で指定した権限を引き継ぎたい

var hoge:* = 1;


コンストラクタで上書きしてもいいが、折角なので宣言でどうにかすると可読性があがるような気がする

  • 型って何よ？

    // BoxクラスはSpriteクラスのサブクラス
var box:Box = new Box();
var sprite:Sprite = Sprite(box);
var box2:Box = Box(sprite);
// BoxクラスにあってSpriteクラスに無いプロパティにアクセス

イベントハンドラをインスタンスメソッドとして定義した場合のスコープの怪 †

ローカル ＞ クラスインスタンス の順にスコープを参照するようになるのはいいとして、
その中で書いた無名関数もその順で参照されるようになる。
例えば

private function onEventHandler(e:FooEvent):void {
   trace(_prop); // この_propは自クラスインスタンスに束縛されている
   (function():void {
       trace(_prop); // <- ここの_propも自クラスインスタンスに束縛されている
   })();
}

イベントハンドラ用関数の引数を設定しないと動かない †

addEventListener用のfooを設定する場合に

addEventListener(Event.ENTER_FRAME, foo);

このように定義したら動かなかった

public function foo():void {}

正しくは以下のように書く必要がある

public function foo(evt:Event):void {}

ステージサイズとは、Flashコンパイル時に指定するサイズFlashウィンドウの幅のこと

stage.stageWidth
stage.stageHeight

で取得できる。
addChildされて無いと取れない点に注意。

↑

thisキーワードが静的スコープにもなれば動的スコープにもなる †

基本的には、this.xxx で記述しても静的なスコープとして定義される。
この場合、使う意味としては、ローカル変数名との名前の衝突回避のため。

ただし、無名関数内ではthisは動的スコープとして認識される。
例えば以下のコード

(function():void {trace(this)})();

この出力は以下のとおりである。

[object global]

※"global"はトップレベルを示すオブジェクトで、JSでいうwindowのようなものっぽい

一方こうすることで、objをスコープとして指定できる。

var foo:Function = function():void {trace(this)};
obj.foo()

callやapplyもある

(function():void {trace(this)}).call(obj);
